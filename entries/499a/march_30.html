March 30, 2022

Although I did not get much done recently due to being on vacation for Spring Break, followed by a week of catching up on assignments for other courses, I have returned, and this week I am beginning to think about a logical structure for the JSON data payloads.

For the JSON structure, I am currently focusing on the structure of a single problem. Each problem will have a name as well as a numerical identifier. In addition, a problem will need to have lists of parameters and/or return values, each of which has its own properties such as its data type. However, since different languages handle typing differently, we will probably need to come up with an internal language-independent way of defining types.

Also, each problem will have a list of test cases, which will take the form of example inputs and outputs. Each test case will also have unit testing code that can be run on the student's code, in various languages, using a virtualization system such as Docker. Finally, the payload may also include a link to the solution code, in various languages once again, as well as a description of the problem for instructors to use as an "answer key". However, these will have to somehow be hidden, so the student cannot view this information.
